# WORPL 쪽지 오토클릭 버전 히스토리

## 0.1.4 - 2026-06-12

- 엑셀 `순번` 컬럼이 모두 1로 저장되던 문제를 수정했습니다.
- 엑셀 저장 시 실제 기록 행 기준으로 1, 2, 3 순번을 다시 부여합니다.
- `키워드 신규만`은 입력한 제목 키워드를 엑셀 시트명으로 사용합니다.
- `신규 전체`는 읽을 수량 기준으로 `신규XX개` 시트명을 사용합니다.
- 저장 파일명을 `YYMMDD_HHMM_WP쪽지_엑셀시트명.xlsx` 형식으로 변경했습니다.
- 최초 대상 건수와 실제 기록 건수가 다르면 앱 완료 메시지와 엑셀 `처리 안내` 시트에 차이 사유를 남깁니다.

## 0.1.3 - 2026-06-12

- 엑셀 시트 이름을 `제목 키워드`로 변경했습니다.
- 읽음 처리 완료 후 `쪽지 읽기 XX건 완료` 메시지를 앱 화면과 알림창에 표시합니다.
- 앱 화면 상단에 현재 프로그램 버전을 표시합니다.
- 설치 및 사용 매뉴얼을 추가했습니다.
- 구버전 설치 파일을 정리할 수 있도록 `dist/old-versions` 폴더를 사용합니다.

## 0.1.2 - 2026-06-12

- 전용 Chrome 창을 닫은 뒤 다시 열 때 발생하던 오류를 수정했습니다.
- 신규 쪽지 판정을 제목 셀의 빨간색 `new` 글씨 기준으로 변경했습니다.
- 일반 `new` 텍스트, 신규 이미지, 굵은 글씨 행이 신규 쪽지로 잘못 잡히지 않도록 테스트를 추가했습니다.

## 0.1.1 - 2026-06-12

- WORPL 받은쪽지함 행에서 작성자가 비어 있던 문제를 수정했습니다.
- WORPL 상단 `쪽지함` 신규 건수가 0이면 읽음 처리를 중단하도록 변경했습니다.
- `신규 전체` 실행 시 읽을 수량을 지정할 수 있는 입력칸을 추가했습니다.
- 쪽지 1건 처리 후 받은쪽지함을 다시 로딩해 새로 도착한 쪽지를 반영하도록 개선했습니다.
- 미리보기와 엑셀 컬럼을 WORPL 받은쪽지함 화면 구조에 맞게 조정했습니다.

## 0.1.0 - 2026-06-11

- Electron 기반 Windows 데스크톱 앱을 처음 추가했습니다.
- 전용 Chrome 실행, 받은쪽지함 미리보기, 자동 클릭 읽음 처리, 엑셀 저장 기능을 추가했습니다.
